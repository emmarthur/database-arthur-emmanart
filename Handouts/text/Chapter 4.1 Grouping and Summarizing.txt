The syntax of the aggregate functions
• AVG(expression)
• SUM(expression)
• MIN(expression)
• MAX(expression)
• COUNT(expression)
• COUNT(*)
AVG(expression) 
Computes the average (arithmetic mean) of all the non-null input values.
SUM(expression)
Computes the sum of the non-null input values.
MIN(expression)
Computes the minimum of the non-null input values. Available for any numeric, string, date/time, or enum type, as 
well as inet, interval, money, oid, pg_lsn, tid, xid8, and arrays of any of these types.
MAX(expression)
Computes the maximum of the non-null input values.
COUNT(expression) 
Computes the number of input rows in which the input value is not null.
COUNT(*) 
Computes the number of input rows. This includes NULL values.

Aggregating Data Using GROUP BY
When you use the GROUP BY clause with aggregate functions, you can 
group results according to the values in one or more columns. This allows us 
to perform operations such as sum() or count() for each state in the 
table or for each type of library agency.
SELECT stabr
FROM chap_09_examples.pls_fy2018_libraries
GROUP BY stabr
ORDER BY stabr;

Filtering an Aggregate Query Using HAVING
To filter the results of aggregate functions, we need to use the 
HAVING clause.
You’ve used WHERE clause for filtering, but aggregate functions, 
such as sum(), can’t be used within a WHERE clause because 
they operate at the row level, and aggregate functions work 
across rows.
The HAVING clause places conditions on groups created by 
aggregating.

SELECT pls18.stabr,
  sum(pls18.visits) AS visits_2018,
  sum(pls17.visits) AS visits_2017,
  sum(pls16.visits) AS visits_2016,
  round( (sum(pls18.visits::numeric) - sum(pls17.visits)) /
    sum (pls17.visits) * 100, 1 ) AS chg_2018_17,
  round( (sum(pls17.visits::numeric) - sum(pls16.visits)) /
    sum (pls16.visits) * 100, 1 ) AS chg_2017_16
FROM chap_09_examples.pls_fy2018_libraries pls18
  JOIN chap_09_examples.pls_fy2017_libraries pls17 
    ON pls18.fscskey = pls17.fscskey
  JOIN chap_09_examples.pls_fy2016_libraries pls16 
    ON pls18.fscskey = pls16.fscskey
WHERE pls18.visits >= 0
  AND pls17.visits >= 0
  AND pls16.visits >= 0
GROUP BY pls18.stabr
HAVING sum(pls18.visits) > 50000000
ORDER BY chg_2018_17 DESC;
