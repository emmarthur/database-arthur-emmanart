Most Basic SELECT Syntax
The asterisk following the SELECT keyword is a wildcard, 
which is like a stand-in for a value: it doesn’t represent 
anything in particular and instead represents everything 
that value could possibly be. Here, it’s shorthand for 
“select all columns.”
It is not really a regex metacharacter.

By the standards of SQL, when you use an asterisk in your 
select_list, the order in which the columns are returned is not

Sorting Data with ORDER BY
By the standards of SQL, if an order to the returned data are 
not identified in the select statement, it can be returned 
in any order. The order of the data may change on 
subsequent select statements.
If you care about the order in which the data are returned, 
you need to specify an ORDER BY clause with an 
order_by_list.

It is not unusual for a column to contain rows with 
duplicate values. In the teachers table, the school 
column lists the same school names multiple times 
because each school employs many teachers.
We can use the DISTINCT keyword as part of a query 
that eliminates duplicates and shows only unique values.

Using LIKE and ILIKE with WHERE
The regular expression meta-characters used in SQL are not 
exactly like the meta-characters you may be accustomed in 
more common regular expressions, from Stephen Kleene.
• Percent sign (%) A wildcard matching zero or more 
characters
• Underscore (_) A wildcard matching just one character
• LIKE 'b%’    -- a ‘b’ followed by one or more chars
• LIKE '%ak%’  -- match anything with a ‘ak’ somewhere
• LIKE '_aker' -- a single character before ‘aker’
• LIKE 'ba_er’ -- match any single character between ba and er

'abc' LIKE 'a%'  --   true
'abc' LIKE ' _b_' --   true
'abc' LIKE 'c'   --   false
'ABC' LIKE 'abc' -- false in PostgreSQL
More LIKE
By default, PostgreSQL performs a case-sensitive string 
comparison.
MySQL and MS SQL Server have case-insensitive behaviors by 
default.

Some Other Things We can do with SELECT
The columns named ?column? in your result set is just ugly. 
You can improve the appearance of your result set by 
renaming the columns with expressions, or any other 
column.
You use the AS syntax to create an alias for the column:

, salary AS "Old Salary"
, salary + 1000 AS "New Salary"
FROM teachers.teachers;

What is this NULL Thingie?
In SQL, null or NULL is a special marker used to indicate that a 
data value does not exist in the database. Introduced by the 
creator of the relational database model, E. F. Codd, SQL NULL 
serves to fulfill the requirement that all true relational database 
management systems (RDBMS) support a representation of 
"missing information and inapplicable information".
https://en.wikipedia.org/wiki/Null_(SQL) 
In SQL, NULL is a marker, not a value. This usage is quite different 
from most programming languages, where a null value of a 
reference means it is not pointing to any object.

Using of NULL in SELECT
The syntax of the WHERE clause with the IS NULL or IS NOT 
NULL comparison is different…
When comparing to NULL, you do not use the = or <> operators.
You use the IS NULL or IS NOT NULL comparisons.
You do NOT use the 
equals operator with 
NULL. You just don’t.
