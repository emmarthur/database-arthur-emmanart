209|Part 4: Transactions — 30 points (6*5 each)
210|Use short explanations and SQL examples where appropriate.
211|10. Define a database transaction in your own words.
213|Answer 10:
214|A database transaction is a group of SQL operations that the database treats as one unit: either all of them take effect together, or none of them do if something goes wrong.
216|11. Explain the difference between COMMIT and ROLLBACK.
218|Answer 11:
219|COMMIT saves all changes made during the transaction so they become permanent. ROLLBACK undoes those changes and returns the database to the state before the transaction started.
